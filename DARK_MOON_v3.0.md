# ЧАСТЬ 1. ОБЩАЯ КОНЦЕПЦИЯ

- **Название игры:** «Тёмная луна»

- **Краткое описание:** Детективная ролевая игра для 10–15 человек на один вечер. Игроки получают роли персонажей, которые были в особняке в ночь убийства хозяина. Их задача — вычислить убийцу через обсуждения, улики и голосования. Бот автоматически выбирает финал на основе оставшихся игроков и найденных улик. В игре есть стратегический элемент: улики можно опубликовать сразу или сохранить для использования в голосовании с бонусом +2 штрафных голоса.

- **Таблица ключевых особенностей:**

| Особенность | Описание |
|---|---|
| **Честные алиби** | Все алиби правдивы. Никто не врёт о своём местонахождении. |
| **Два типа козырей** | Начальный (личная тайна, вскрывается при исключении) и найденный (улика из QR, можно использовать в голосовании). |
| **Автоматическое вскрытие** | Начальные козыри вскрываются при исключении игрока. |
| **Улики из реальности** | 15 QR-кодов разложены по комнате. При сканировании бот выдаёт случайную улику. |
| **Улика → козырь** | Игрок может опубликовать улику сразу или сохранить для использования в голосовании с бонусом +2 штрафных голоса. |
| **Голосование через бота** | Двухфазное: обсуждение (с кнопкой согласия в ЛС) → фаза выбора (инлайн-кнопки в группе, 2 мин, авто-подсчёт). Честное, анонимное, с возможностью смены голоса. |
| **Полный контроль ведущего** | Ведущий управляет только назначением ролей, стартом игры и финалом. Голосования запускаются игроками. |
| **Автоматический финал** | Бот сам выбирает финал на основе оставшихся игроков и найденных улик. |
| **Лимит улик** | Один игрок может найти не более 3 улик. |

- **Время убийства:** **22:15** (фиксированное). Все алиби привязаны к этому времени.

# ЧАСТЬ 2. РОЛИ УЧАСТНИКОВ

## 2.1. Ведущий (админ)

Это вы. Вы не играете, а управляете процессом.

**Ваши задачи:**
- Раздать роли через бота (вручную, подбирая под человека).
- Объяснить правила перед игрой.
- Запустить игру командой `/startgame`.
- Запустить финал командой `/final` (когда останется ≤3 игроков).
- Управлять перерывами на тосты и еду (устно).

**Ваши команды боту:**

| Команда | Что делает |
|---|---|
| `/assign @username Роль` | Назначает роль игроку (вместе с начальным козырем) |
| `/startgame` | Отправляет приветствие всем игрокам, переводит игру в фазу обсуждения |
| `/final` | Запускает финальное раскрытие (бот выбирает финал автоматически) |
| `/status` | Показывает состояние игры: фаза, активные игроки, найденные улики |

> **Важно:** Команды `/vote` и `/results` **больше не используются**. Голосование инициируется игроками через кнопку в личных сообщениях бота.

## 2.2. Игроки

Это ваши гости. Каждый получает роль и участвует в расследовании.

**Задачи игроков:**
- Озвучить своё алиби вслух.
- Искать QR-коды с уликами.
- Обсуждать улики и подозреваемых за столом.
- Решать: опубликовать улику или сохранить как козырь.
- Инициировать голосование (через кнопку в ЛС бота).
- Голосовать за исключение подозреваемых.
- Использовать сохранённые улики во время голосования через `/use`.

**Команды игроков боту:**

| Команда | Что делает |
|---|---|
| `/start` | Зарегистрироваться в боте (также обрабатывает QR-коды: `/start qr`) |
| `/publish` | Опубликовать найденную улику в общий чат |
| `/use` | Активировать сохранённую улику как козырь **во время фазы голосования** (добавляет +2 штрафных голоса цели) |
| `/myclues` | Показать свои сохранённые улики |
| `/info` | Показать свою роль и начальный козырь |

# ЧАСТЬ 4. СИСТЕМА УЛИК

## 4.1. Определение

Улики — это информация, которую игроки находят в реальном мире через QR-коды.

## 4.2. Количество QR-кодов: 15 штук.

## 4.3. Расположение QR-кодов

Разложены по комнате в разных местах (под вазой, на холодильнике, в книге и т.д.).

## 4.4. Механика сканирования QR-кода

1. Все 15 QR-кодов ведут на **одну ссылку**: `t.me/YourBot?start=qr`.
2. При сканировании бот проверяет роль игрока и выдаёт **случайную улику** из списка.
3. Улика **удаляется из списка** — её больше никто не найдёт.
4. Один игрок может найти **не более 3 улик**.
5. В общий чат бот пишет: *«[Имя] нашёл(ла) улику!»* (но не её содержание).
6. Если все 15 улик найдены, при сканировании бот пишет: *«Все улики уже найдены.»*

## 4.6. Правила работы с уликой (опубликовать / сохранить)

| Вариант | Действие | Результат |
|---|---|---|
| **Опубликовать** | Написать боту `/publish` | Улика появляется в общем чате. Все её видят. Улика удаляется у игрока. |
| **Сохранить как козырь** | Ничего не делать | Улика остаётся у игрока. Можно использовать во время голосования через `/use`. |

## 4.7. Использование сохранённой улики (`/use`)

- Команда `/use` доступна **только во время фазы голосования** (не во время обсуждения).
- Игрок пишет `/use` в личку боту.
- Бот **публикует улику в общий чат**.
- Если улика указывает на конкретного игрока (см. таблицу 4.5), **этот игрок получает +2 штрафных голоса**.
- Если улика не указывает на конкретного человека, она просто публикуется без бонуса.
- Улику можно использовать **только один раз**.

## 4.8. Случайная выдача улик

- Все 15 QR-кодов ведут на одну ссылку.
- При сканировании бот выдаёт **случайную улику** из числа ещё не найденных.
- Улика удаляется из списка и становится недоступной для других.
- Один игрок может найти несколько улик (но не более 3).

## 4.9. Лимит улик на игрока

- Один игрок может найти **не более 3 улик**.
- Если игрок уже нашёл 3 улики, при сканировании QR-кода бот пишет: *«Вы уже нашли 3 улики. Дайте другим шанс!»*
- Улика остаётся доступной для других игроков.

## 4.10. Передача улик

Улику **нельзя передавать** другому игроку. Она привязана к тому, кто её нашёл.

## 4.11. Судьба улик при исключении

Если игрок исключён, его **найденные улики НЕ вскрываются** (в отличие от начальных козырей). Они исчезают вместе с ним.

# ЧАСТЬ 5. СИСТЕМА ГОЛОСОВАНИЯ (ДВУХФАЗНАЯ)

> **Ключевое изменение версии 3.0:** Голосование больше не управляется командами `/vote` и `/results`. Вместо этого используется двухфазная система: **фаза обсуждения** → **фаза голосования**.

## 5.1. Фазы игры

Игра находится в одной из следующих фаз:

| Фаза | Описание | Длится |
|---|---|---|
| `setup` | Назначение ролей, подготовка | До `/startgame` |
| `discussion` | Игроки обсуждают, ищут улики, могут инициировать голосование | Неограниченно |
| `voting` | Идёт голосование: выбор цели на исключение | 2 минуты |
| `finished` | Игра окончена, финал показан | — |

## 5.2. Фаза обсуждения (discussion)

- Начинается после команды ведущего `/startgame`.
- Игроки обсуждают улики, алиби, подозреваемых — в групповом чате или устно за столом.
- В личных сообщениях бота у каждого активного игрока есть кнопка: **«Начать голосование»**.

**Механика кнопки «Начать голосование»:**
1. Любой активный игрок нажимает кнопку в ЛС бота.
2. Бот регистрирует согласие этого игрока.
3. Каждому активному игроку при обновлении показывается: *«X из Y игроков согласны начать голосование»*.
4. Когда количество согласившихся превышает **50% от числа активных игроков**, бот автоматически переключает игру в фазу `voting`.

**Реакция бота при переходе в фазу голосования:**
- Бот отправляет в групповой чат: *«🗳️ Большинство игроков согласно. Голосование начинается! У вас 2 минуты.»*
- Бот создаёт сообщение с **инлайн-кнопками** (имена всех активных игроков, по 3–4 в ряд).
- Запускается таймер на 2 минуты.

## 5.3. Фаза голосования (voting)

- Каждый активный игрок нажимает на кнопку с именем того, кого хочет исключить.
- Игрок может **сменить голос**, нажав на другую кнопку.
- Голосование **анонимное** (количество голосов не показывается).
- Ведущий **исключён** из списка для голосования.

**Завершение голосования (автоматическое):**

Голосование завершается в одном из двух случаев:
1. **Истекли 2 минуты** — бот автоматически подсчитывает результаты.
2. **Все активные игроки проголосовали** — бот подсчитывает результаты досрочно.

**Подсчёт результатов:**
1. Бот собирает все голоса (учитывается **последний** голос каждого игрока).
2. Бот добавляет **штрафные голоса** от использованных улик (`/use`).
3. Бот проверяет **явку** (>50% активных игроков должно проголосовать).
4. Определяется игрок с наибольшим количеством голосов.

## 5.4. Результаты голосования

**Если явка < 50% и не все голоса за одного:**
> *«Явка меньше 50%. Голосование недействительно. Возвращаемся к обсуждению.»*
→ Фаза переключается обратно на `discussion`.

**Если ничья (два+ игрока набрали одинаково):**
> *«Ничья! Никто не исключён. Начинается новый раунд обсуждения.»*
→ Фаза переключается обратно на `discussion`.

**Если все проголосовали за одного (даже при явке < 50%):**
→ Исключение происходит автоматически (единогласное решение).

**Если есть явный лидер (явка > 50%):**
1. Бот объявляет: *«🗳️ Исключается [Имя]!»*
2. Бот проверяет начальный козырь исключённого (`alibi_protected = True`).
3. Если есть — публикует: *«🔍 Вскрыт козырь [Имя]: [текст начального козыря]»*.
4. Игрок деактивируется (становится наблюдателем).
5. Штрафные голоса очищаются.
6. Если активных игроков осталось **больше 3** → фаза переключается на `discussion`, бот пишет: *«Новый раунд обсуждения. Игроки, вы можете инициировать голосование через кнопку в ЛС.»*
7. Если активных игроков осталось **3 или меньше** → бот пишет: *«Осталось 3 игрока. Ведущий, используйте /final для завершения игры.»*

## 5.5. Использование `/use` во время голосования

- В любой момент фазы `voting` игрок может написать `/use` в ЛС бота.
- Бот публикует улику в общий чат.
- Если улика указывает на цель — цель получает +2 штрафных голоса.
- Штрафные голоса не отображаются для других игроков, но учитываются при подсчёте.

## 5.6. Статус исключённого игрока

- Выбывает из игры.
- Его **начальный козырь вскрывается** (если ещё не был).
- Его **найденные улики НЕ вскрываются**.
- Может наблюдать за игрой, но не участвует в обсуждении и голосовании.

# ЧАСТЬ 6. ПОЛНЫЙ ХОД ИГРЫ

## 6.1. Подготовка (до вечеринки)

1. Создать бота в Telegram (получить токен).
2. Настроить переменные окружения: `BOT_TOKEN`, `ADMIN_ID`, `GROUP_CHAT_ID`.
3. Сгенерировать **15 QR-кодов** (все ведут на `t.me/YourBot?start=qr`).
4. Распечатать 15 копий QR-кода.
5. Разложить 15 QR-кодов по комнате в разных местах.
6. Подготовить список ролей с биографиями для каждого персонажа.

## 6.2. Начало вечера (за столом)

1. Гости собираются, тосты.
2. Ведущий просит всех запустить бота (`/start`).
3. Ведущий назначает роли через бота (`/assign @username Роль`).
4. Ведущий объясняет правила (убийство в 22:15, алиби, улики, голосование).

## 6.3. Озвучивание алиби

Каждый игрок по кругу встаёт и говорит своё алиби.

## 6.4. Запуск игры

Ведущий пишет `/startgame`. Бот отправляет приветствие всем игрокам. Начинается фаза `discussion`.

## 6.5. Поиск QR-кодов (10–15 минут)

- Все встают и ищут QR-коды (15 штук, разложены по комнате).
- При сканировании бот выдаёт случайную улику в личку.
- Игроки могут публиковать улики (`/publish`) или сохранять для использования в голосовании (`/use`).
- Лимит: не более 3 улик на игрока.

## 6.6. Раунды обсуждения и голосования

1. Все садятся за стол.
2. Фаза `discussion`: обсуждают улики, алиби, подозреваемых.
3. Любой игрок может нажать кнопку «Начать голосование» в ЛС бота.
4. Когда >50% согласны → фаза `voting` (2 минуты).
5. Игроки голосуют кнопками в групповом чате.
6. Авто-подсчёт: исключение или возврат к обсуждению.
7. Повтор, пока не останется ≤3 игроков.

## 6.7. Перерывы

Ведущий объявляет перерывы устно (между раундами для тостов и еды).

## 6.8. Финал

Когда остаётся **≤3 игроков**, ведущий пишет `/final`. Бот **автоматически выбирает** финал на основе оставшихся игроков и найденных улик (см. Часть 7).

## 8.8. Логика обработки команд

### `/start` и QR-код
```
1. Пользователь пишет /start или /start qr
2. Если /start:
   - Зарегистрировать пользователя в БД
   - Отправить: «Добро пожаловать! Ожидайте назначения роли.»
3. Если /start qr:
   - Проверить наличие роли (иначе: «Вам ещё не назначена роль.»)
   - Проверить остались ли улики (иначе: «Все улики уже найдены.»)
   - Проверить лимит 3 улики (иначе: «Вы уже нашли 3 улики. Дайте другим шанс!»)
   - Выбрать случайную улику из доступных
   - Отметить как найденную, сохранить у игрока
   - В личку: «🔍 Вы нашли улику: [текст]»
   - В группу: «[Имя] нашёл(ла) улику!»
```

### `/assign @username Роль`
```
1. Проверить, что отправитель = ADMIN_ID
2. Извлечь @username и текст роли
3. Найти игрока в БД по username
4. Сохранить роль и начальный козырь
5. В личку игроку: «Ваша роль: [Роль]. Ваш начальный козырь: [Козырь]»
6. В группу: «[Имя] получил(а) роль!»
```

### `/startgame`
```
1. Проверить ADMIN_ID
2. Установить phase = 'discussion' в games
3. Увеличить round до 1
4. Разослать всем активным игрокам: «Игра началась! Фаза обсуждения.»
5. Каждому игроку в ЛС: отправить кнопку «Начать голосование»
6. В группу: «Игра «Тёмная луна» началась! Обсуждайте, ищите улики.»
```

### Кнопка «Начать голосование» (в ЛС)
```
1. Игрок нажимает кнопку (callback_data = "consent_yes")
2. Проверить phase == 'discussion'
3. Сохранить/обновить consent_votes[player_id] = True
4. Посчитать количество согласившихся
5. Если count > alive_count / 2:
   - Переключить phase = 'voting'
   - Увеличить round
   - Отправить в группу: «🗳️ Большинство согласно! Голосование: 2 минуты.»
   - Создать инлайн-клавиатуру с именами активных игроков
   - Запустить таймер 120 секунд (job_queue.run_once)
6. Иначе: обновить счётчик согласий для всех кнопок
```

### Голосование (кнопки в группе)
```
1. Игрок нажимает кнопку (callback_data = "vote_{player_id}")
2. Проверить phase == 'voting'
3. Проверить, что игрок активен
4. Сохранить/перезаписать votes[voter_id] = target_id
5. Ответить: «Ваш голос учтён. Вы можете изменить его.»
6. Если проголосовали все активные — досрочный подсчёт
```

### Авто-подсчёт (по таймеру или при all_voted)
```
1. Собрать все голоса из votes
2. Добавить штрафные голоса из penalty_votes
3. Проверить явку: len(votes) > alive_count * 0.5
4. Если явка < 50% и не все за одного:
   → «Явка < 50%. Голосование недействительно.» → phase = discussion
5. Если ничья (max_count дублируется):
   → «Ничья! Никто не исключён.» → phase = discussion
6. Иначе:
   - Определить исключённого (max голосов)
   - Скопировать голоса в votes_history
   - Объявить исключение в группу
   - Если alibi_protected: опубликовать initial_clue
   - Деактивировать игрока
   - Очистить penalty_votes, consent_votes, votes
   - Если alive > 3: phase = discussion + кнопки в ЛС
   - Если alive <= 3: уведомить ведущего о /final
```

### `/use` (во время голосования)
```
1. Проверить phase == 'voting'
2. Проверить наличие улик у игрока
3. Взять первую улику из clues_found[0]
4. Опубликовать в группу: «🔍 [Имя] использует улику: [текст]»
5. Если CLUE_TO_TARGET[clue_id] существует:
   - target_id = найти игрока по роли
   - penalty_votes[target_id] += 2
6. Удалить улику из clues_found игрока
7. Ответить игроку подтверждение
```

### `/final`
```
1. Проверить ADMIN_ID
2. Собрать список ролей активных игроков (remaining_roles)
3. Собрать список ID найденных улиц (found_clue_ids)
4. Выбрать финал по приоритету (см. 7.1)
5. Определить killer_id (цель финала)
6. Если killer_id:
   - Найти в votes_history всех, кто голосовал за killer_id
   - Собрать уникальные имена победителей
   - Добавить строку с победителями в финал
7. Отправить финал в групповой чат
8. Установить phase = 'finished'
```

- [ ] Создать бота в Telegram (через @BotFather)
- [ ] Записать токен бота
- [ ] Добавить бота в групповой чат как администратора
- [ ] Узнать свой `ADMIN_ID` и `GROUP_CHAT_ID`
- [ ] Настроить переменные окружения (.env)
- [ ] Запустить бота на сервере
- [ ] Сгенерировать 15 QR-кодов (все на `t.me/YourBot?start=qr`)
- [ ] Распечатать 15 копий QR-кода
- [ ] Разложить QR-коды по комнате
- [ ] Подготовить биографии персонажей (карточки)

## 9.2. Во время игры (пошаговый сценарий)

1. [ ] Все собрались, тосты
2. [ ] Попросить всех написать `/start` боту в ЛС
3. [ ] Назначить роли: `/assign @username Анна — вдова`
4. [ ] Объяснить правила устно
5. [ ] Все озвучивают алиби (по кругу)
6. [ ] Написать `/startgame` — игра началась!
7. [ ] Поиск QR-кодов (10–15 минут)
8. [ ] Раунд 1: обсуждение → голосование (игроки нажимают кнопку в ЛС)
9. [ ] Перерыв (по желанию)
10. [ ] Раунд 2: обсуждение → голосование
11. [ ] Перерыв (по желанию)
12. [ ] Раунд 3: обсуждение → голосование
13. [ ] Повторять, пока не останется ≤3 игроков
14. [ ] Написать `/final` — финал!
15. [ ] Финальные тосты, обсуждение

## 9.3. Скрипт для ведущего (вступительная речь)

> «Добро пожаловать в особняк «Тёмная луна». Сегодня вечером мы расследуем убийство хозяина дома. Все вы — гости, которые были здесь в ночь убийства.
>
> Ваша задача — выяснить, кто убийца. У вас есть алиби — оно правдиво. Есть QR-коды с уликами. Есть голосования.
>
> Я — ваш ведущий. Я назначаю роли и запускаю игру. Голосования вы инициируете сами через кнопку в боте.
>
> Удачи. И помните: правда всегда сложнее, чем кажется.»

# СПИСОК ИЗМЕНЕНИЙ (2.2 → 3.0)

| № | Что изменено | Краткое описание |
|---|---|---|
| 1 | **Система голосования** | Полностью переработана. Вместо команд `/vote`/`/results` — двухфазная система: обсуждение (кнопка согласия в ЛС) → голосование (инлайн-кнопки в группе, 2 мин, авто-подсчёт). |
| 2 | **Фазы игры** | Добавлены фазы: `setup`, `discussion`, `voting`, `finished`. |
| 3 | **Таблица consent_votes** | Новая таблица БД для отслеживания согласий на голосование. |
| 4 | **Таймер голосования** | Добавлен 2-минутный таймер с авто-подсчётом. |
| 5 | **Досрочное завершение** | Если все активные игроки проголосовали — подсчёт до истечения таймера. |
| 6 | **`/vote` и `/results` удалены** | Эти команды больше не используются. |
| 7 | **`alibi_protected` уточнён** | Флаг: начальный козырь ещё не вскрыт. Публикуется при исключении. |
| 8 | **Поле `clue_used` удалено** | Заменено на `alibi_protected`. |
| 9 | **Обновлён чек-лист ведущего** | Убраны шаги с `/vote` и `/results`. |
| 10 | **Обновлён список команд** | У админа остались: `/assign`, `/startgame`, `/final`, `/status`. |
